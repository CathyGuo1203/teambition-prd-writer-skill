## Teambition 产品核心概念

### 核心模块

Teambition 的核心功能围绕以下模块构建：

| 模块 | 描述 |
| --- | --- |
| **项目 (Project)** | 管理工作的核心空间，可容纳任务、文件、日程等。 |
| **任务 (Task)** | 项目中最基本的工作单元，可分配给成员、设置截止日期和优先级。 |
| **任务字段 (Task Fields)** | 可自定义任务属性，用于分类、筛选和跟踪任务。 |
| **看板 (Kanban)** | 以卡片和列的形式可视化任务流程，直观展示工作进展。 |
| **甘特图 (Gantt Chart)** | 以时间轴的形式展示任务计划和依赖关系，用于项目排期。 |
| **工作流 (Workflow)** | 定义任务在不同阶段之间的流转规则，实现流程自动化。 |
| **自动化 (Automation)** | 根据预设规则自动执行重复性操作，提升团队效率。 |
| **成员 (Members)** | 项目的参与者，拥有不同的角色和权限。 |
| **权限 (Permissions)** | 控制成员对项目内容和功能的操作权限。 |

### 目标用户与行业

Teambition 主要服务于以下行业的中小企业团队：

*   **互联网软件**
*   **智能硬件开发**
*   **金融软件**
*   **半导体/芯片**
*   **汽车制造**

覆盖的团队类型包括：

*   **研发团队**
*   **交付团队**
*   **市场团队**
*   **设计团队**

### 功能详述

#### 项目 (Project)

*   **项目模板:** 提供超过 40 种项目模板，覆盖研发、市场、设计等多种场景。
*   **多种视图:** 支持看板、列表、甘特图、日历等多种视图切换。
*   **项目群:** 用于管理多个关联项目。

#### 任务 (Task)

*   **子任务:** 支持将复杂任务拆解为多层级的子任务。
*   **任务依赖:** 可设置任务之间的前后置依赖关系。
*   **自定义字段:** 支持文本、数字、日期、成员、下拉菜单等多种字段类型。
*   **评论与附件:** 可在任务中进行讨论，并上传文件、图片等附件。
*   **任务提醒:** 支持自定义任务提醒规则。

#### 工作流与自动化

*   **自定义工作流:** 可根据团队的实际流程，自定义任务在看板上的流转阶段。
*   **自动化规则:** 可设置触发器和动作，例如"当任务进入'已完成'阶段时，自动通知相关人员"。
